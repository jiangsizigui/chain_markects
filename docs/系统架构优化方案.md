# 预测市场平台系统架构优化方案

## 1. 总体目标

- 构建多交易机器人驱动的高频模拟交易环境，持续产生成交、盘口与行为数据
- 升级市场端为接近真实交易平台的交互体验（价格走势、K线、盘口、实时变化）
- 构建类似 SPSS/Tableau 的数据分析工作站，支持数据集管理与多模型分析
- 打通全链路数据闭环：交易产生数据 -> 数据存储 -> 数据处理 -> 分析展示

## 2. 分层架构

### 2.1 交易与仿真层

- Matching Engine：订单撮合与成交生成
- Bot Orchestrator：机器人生命周期管理（启停、参数、策略）
- Bot Strategy Runtime：做市、趋势、噪声等策略执行
- Market Data Aggregator：盘口快照、Ticker、OHLCV 聚合

### 2.2 数据中台层

- OLTP：markets/orders/trades/positions/wallets/bots
- ODS：trade_events/order_events（原始事件）
- DWS：fact_trades_1m/fact_user_behavior_daily 等宽表
- Feature Service：相关性、波动率、行为特征、时序特征

### 2.3 分析与展示层

- Analytics Workstation：数据集管理、字段识别、筛选、模型分析
- Visualization Studio：折线图、柱状图、热力图、K线图、透视视图
- AI Copilot：对统计结果给出摘要、风险、建议

## 3. 数据流闭环设计

1) 用户与机器人下单/成交 -> 写入 `orders`/`trades`
2) 行情聚合服务将成交流转为 `ticker/depth/ohlcv`
3) 分析引擎读取平台数据或上传数据形成分析任务
4) Python 分析引擎输出结构化结果（8模型）
5) AI 接口基于模型结果生成解释建议
6) 前端工作站统一展示并可导出报告

## 4. 核心功能模块划分

### 4.1 机器人管理模块（Admin）

- 机器人列表：名称、策略、状态、强度、更新时间
- 机器人操作：新建、启动、停止、单次运行
- 模拟交易控制：批量生成模拟交易

### 4.2 市场交易终端模块（Marketplace）

- 盘口与买卖交互：限价/市价、下单、撤单
- 实时数据更新：轮询/后续可扩展 WebSocket
- K线与走势：OHLCV API + 图表展示

### 4.3 数据分析工作站模块（Analytics）

- 数据源选择：平台数据 / 上传数据 / 混合数据
- 拖拽上传：JSON/CSV/XLSX 自动识别
- 模型选择：ARIMA、回归、贝叶斯、情绪、KMeans、GARCH、偏差、相关性
- AI 辅助分析：结果摘要、风险点、可执行建议

## 5. API 设计（当前与扩展）

### 已实现（本阶段）

- `POST /api/admin/bootstrap/simulate-trades`
- `GET /api/admin/bots`
- `POST /api/admin/bots`
- `PATCH /api/admin/bots/:id`
- `POST /api/admin/bots/:id/start`
- `POST /api/admin/bots/:id/stop`
- `POST /api/admin/bots/run-once`
- `GET /api/markets/:marketId/ohlcv`
- `POST /api/analyze`
- `POST /api/analyze/ai`

### 建议扩展

- `GET /api/markets/:id/ticker`
- `GET /api/markets/:id/depth`
- `WS /ws/markets/:id`（行情与成交流）
- `POST /api/datasets/upload`
- `GET /api/datasets/:id/schema`
- `POST /api/analysis/jobs`

## 6. 数据结构建议

- `bots`: id/name/strategy/enabled/marketIds/intensity/maxOrderSize
- `ohlcv`: marketId/tf/ts/open/high/low/close/volume/trades
- `analysis_jobs`: datasetId/model/params/status/progress/resultRef
- `analysis_reports`: jobId/summary/charts/aiInsight

## 7. 实施路线图

- Phase 1（已落地）：机器人管理 MVP、模拟交易、K线 API、工作站与 AI 辅助分析
- Phase 2：数据集注册中心、字段画像、分析任务队列、可视化编排器
- Phase 3：WebSocket 实时总线、风控规则、策略回测、报告系统
